  • For Computer Science Beginners

    Welcome to Computer Science!

    CSC226 is your first step into the world of computer science! This class introduces you to software design using the Python programming language. But what exactly does that mean?

    What is a Programming Language?

    Just like humans use languages (English, Spanish, etc.) to communicate with each other, we use programming languages to communicate with computers. Python is one of these languages, and it's known for being especially friendly for beginners! Think of it as learning to write instructions that your computer can understand and follow.

    Your Coding Workspace: PyCharm

    In this class, we'll use something called PyCharm, which is what we call an IDE (Integrated Development Environment). If this sounds complicated, think of it like Microsoft Word, but for coding:

    • Just like Word helps you write documents by catching spelling mistakes, PyCharm helps you write code by catching coding mistakes
    • Word gives you tools like formatting and spell-check; PyCharm gives you tools like code completion and debugging
    • Word shows you when you've made grammar errors; PyCharm shows you when you've made syntax errors in your code

    Key Things to Know

    • Don't be afraid to make mistakes! Making errors and fixing them is a normal part of learning to code
    • Practice is crucial: Programming is like learning a new language - the more you practice, the better you get
    • Ask questions: Computer science has its own vocabulary, so don't hesitate to ask when you don't understand a term
    • AI is a fantastic tool, if used correctly! This class is centered around teaching you the fundamentals of algorithmic thinking, so an understanding of the basics is crucial

  • Git & GitHub

    What are Git and GitHub?

    Imagine you're working on a big group project where everyone needs to contribute to a document. Now imagine if you could:

    • Track every change made to the document
    • Know who made each change and when
    • Go back to any previous version if something goes wrong
    • Work on your own copy without affecting others

    Git: Your Local Version Control

    Git is a tool that runs on your computer (like Microsoft Word or PyCharm). It:

    • Keeps track of all changes to your code files
    • Lets you save different versions of your work
    • Works on your personal computer, even without internet
    • Helps you experiment with changes without breaking your working code

    GitHub: Your Online Code Storage

    GitHub is like Google Drive, but specifically designed for code. It:

    • Stores your code online so you can access it from anywhere
    • Lets you share your code with others
    • Makes it easy for multiple people to work on the same project
    • Keeps a backup of your code in case something happens to your computer

    How They Work Together

    Think of it this way:

    • Git is like working on files saved on your computer - you make changes locally first
    • GitHub is like Google Drive - once you're ready to share your changes, you upload (or "push") them to GitHub where others can see and use them
    • When you want to work on code that's on GitHub, you download (or "pull") it to your computer first, then use Git to track your changes

    We'll be using both tools throughout the course to manage and share our code!
